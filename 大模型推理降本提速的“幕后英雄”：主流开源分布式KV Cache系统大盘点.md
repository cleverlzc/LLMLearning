# 大模型推理降本提速的“幕后英雄”：主流开源分布式KV Cache系统大盘点
> 原创 一研
> 发布时间：2026-06-07 21:09:00
> 关键词：分布式 KVCache、分离式架构、缓存池化、分层存储、LLM 推理优化、PagedAttention、Prefill-Decode 解耦、全局缓存复用、vLLM、SGLang、Mooncake、LMCache、FlexKV

在大模型在线推理场景中，性能瓶颈往往不在于 GPU 算力，而在于KVCache 显存资源的低效利用与重复计算。

在Transformer自回归生成中，KV Cache通过缓存历史Token的Key-Value矩阵，避免了重复计算——这既是推理加速的核心，也是“吃显存”的大头。一个长序列或高并发请求，轻轻松松就能占满昂贵的HBM。更棘手的是：每一次新请求到来，即使系统里已经有人算过相同的前缀，传统方案依然要从头计算一遍。重复计算+显存浪费=成本翻倍、延迟恶化。

传统单机推理架构下，KV 缓存与推理实例强绑定，仅能在单卡、单会话生命周期内生效。同源提示词反复触发 Prefill 计算、负载均衡打散集群缓存、显存碎片化严重、长上下文吞吐受限等问题，持续推高推理成本、制约服务并发能力。

为解决上述痛点，LLM 推理架构已完成从「单卡本地缓存」到「单机分页缓存」向分布式解耦 KVCache的全面升级，并已成为业界共识。通过构建全局缓存资源池，彻底打破单机显存容量壁垒，实现跨节点、跨实例、跨会话的缓存复用，成为当前大规模、低成本、高稳定大模型推理的核心基建。通过分层存储、分离式架构等手段，把缓存从孤立孤岛变成了可流动、可复用的共享资源，直接带来首Token延迟骤降、吞吐翻倍和GPU成本显著降低。

本文结合 2025–2026 年最新开源生态与工业落地实践，系统梳理六大主流分布式 KVCache 体系，完整拆解技术架构、核心特性、差异化优势与适用场景，为企业生产级架构选型提供权威、可落地的参考依据。

## 一、KVCache 技术演进：从单机受限到全局池化
纵观行业发展，KVCache 技术历经四代迭代，每一次架构升级都精准解决上一代的核心短板，逐步实现从“可用”到“高效可规模化”的进阶。

### 第一代：会话独占缓存（原始架构）
早期推理架构无任何缓存复用机制，每轮对话请求均独立生成 KV 张量，会话结束后缓存资源即刻释放。该模式存在大量无效重复计算，Prefill 算力浪费严重，无法支撑高并发线上业务。

### 第二代：单机分页缓存（vLLM PagedAttention）
vLLM 首创 PagedAttention 分页缓存机制，将 KV 缓存切分为固定尺寸的 Block 单元，实现精细化显存分页管理，彻底解决显存碎片问题，将显存利用率提升至 90% 以上。该方案的局限性在于，缓存资源仅局限于单机实例，无法在集群环境下跨节点共享。

### 第三代：单机前缀缓存（SGLang Radix / vLLM APCA）
基于前缀树索引能力，单机架构可实现相同前缀请求的 KV 缓存复用，有效减少重复 Prefill 计算、提升单机吞吐。但在分布式集群部署中，负载均衡机制会打散同源请求，导致前缀缓存完全失效，无法支撑规模化集群落地。

### 第四代：分布式解耦 KVCache（工业主流架构）
当前行业标准化架构，核心设计是将缓存层与推理计算层物理解耦，独立构建全局统一缓存资源池。集群内 GPU 显存、CPU 内存、NVMe 磁盘等异构存储资源统一调度，实现全集群 KV 缓存共享复用，从架构根源上解决重复计算、显存不足、集群缓存失效等核心问题，是长上下文、高并发、低成本推理的关键底座。

## 二、主流开源分布式 KVCache 系统全景解析
目前工业界成熟的开源分布式 KVCache 生态可分为两大体系：
1. 专注缓存调度与存储的独立分布式缓存中间件：Mooncake、LMCache、FlexKV
2. 聚焦推理服务与集群编排的引擎生态框架：NVIDIA Dynamo、vLLM、SGLang

> 说明：后者严格意义上讲不是专门的KVCache系统，但KVCache无法脱离推理框架独立落地；主流推理框架均原生支持分布式KV Cache管理，能力成熟，故纳入统一对比，形成全局选型视角。

### 1. Mooncake｜工业级解耦式分布式 KVCache 标杆
**项目归属**：月之暗面（Kimi）与清华大学MADSys实验室联合（FAST’25 最佳论文项目，业界标准解耦 KV 存储系统）
**产品定位**：面向大规模 LLM 推理场景设计的计算缓存解耦架构，是目前线上落地规模最大、稳定性最强的分布式 KVCache 系统之一，已支撑 Kimi 等大规模商用业务稳定迭代。

Mooncake的核心技术是采用以键值缓存（KVCache）为中心的分离式架构，通过存储与计算解耦，优化大模型推理的吞吐量和延迟。其核心架构主要由全局调度器（Conductor）、Prefill节点集群、Decode节点集群、分布式KVCache池以及RDMA通信组件（Messenger）构成。

#### 核心架构：标准 Prefill-Decode 物理分离架构
通过集群角色物理拆分，实现计算、解码、缓存资源隔离调度，避免资源争抢：
- **Prefill 计算集群**：专注 Token 预填充与 KV 张量生成，承载高算力消耗的预计算任务；
- **Decode 推理集群**：仅负责解码推理逻辑，通过远程拉取全局缓存池 KV 数据，无需重复执行 Prefill 计算；
- **独立分布式缓存集群**：专职托管全集群 KV Block 资源，不承载推理计算任务，专注缓存存储、寻址与冷热调度。

#### 核心技术能力
1. 采用 GPU/DRAM/NVMe 三级分层存储架构，支持缓存冷热自动迁移，兼顾极致访问性能与大容量存储能力；
2. 搭载全局 Radix 前缀树与 Block 路由表，实现精准跨节点寻址与最长前缀匹配，最大化全局缓存复用率；
3. 原生支持 RDMA/GDS 零拷贝硬件直通传输，不占用 GPU 计算算力，大幅降低跨节点数据传输延迟；
4. 基于全局引用计数与 LRU 淘汰策略，保障多会话共享缓存的安全性与资源高效利用率。

#### 适配场景与技术特点
Mooncake 架构标准化程度高、集群稳定性强，大规模场景下缓存命中率表现优异，经过海量线上业务验证。
短板：依赖 RDMA 专用网络组网，硬件部署成本较高，轻量化小集群部署性价比偏低。
适用：万卡级超大算力集群、高并发在线对话、超长上下文推理、云厂商公共推理服务等对性能与稳定性要求极高的生产场景。

> 名词注解：解耦式/分离式同义；分离式=存算分离，缓存独立部署池化；分布式=缓存跨多节点分布协同；Mooncake 同时具备分离式+分布式特性。

开源地址：https://github.com/kvcache-ai/Mooncake

### 2. LMCache｜轻量化无侵入全引擎兼容分布式缓存
**项目归属**：UC Berkeley + TensorMesh 联合开源
**产品定位**：轻量化插件式分布式 KVCache 中间件，主打无侵入接入、全推理引擎兼容，是中小企业存量推理集群低成本升级分布式缓存能力的首选方案。

#### 核心技术特性
1. 构建 L1 本地显存、L2 集群远端内存、L3 远端磁盘三级缓存体系，逐层平衡访问速度与存储容量；
2. 支持 TCP/RDMA 自适应传输切换，无强制硬件依赖，普通服务器集群可快速部署落地；
3. 采用可插拔架构设计，灵活适配 NVMe、Redis、Alluxio 等多种存储后端，适配不同业务架构体系；
4. 支持缓存持久化存储，服务重启后固定提示词缓存不丢失，持续提升长期服务缓存命中率。

LMCache 核心能力聚焦分布式块缓存管理与存储卸载，自身不实现前缀匹配逻辑，缓存前缀复用能力依托 vLLM、SGLang 等上层推理引擎实现，架构轻量化、职责边界清晰。

#### 适配场景
存量 vLLM 集群改造、中小规模推理集群、成本敏感型业务、企业知识库固定提示词问答场景。

开源地址：https://github.com/LMCache/LMCache

### 3. FlexKV｜腾讯云 TACO 云原生分布式 KVCache
**项目归属**：腾讯云 TACO 团队（2026 年正式开源）
**产品定位**：面向 K8s 云原生场景设计的多级异构分布式 KVCache 系统，核心价值是充分盘活集群闲置算力与存储资源，适配云原生弹性扩缩容的业务特性。

#### 官方标准四层分层架构
本地GPU显存 → 本机DRAM → 远端集群DRAM → 远端NVMe，通过分层冷热调度机制，平衡推理访问性能与资源使用成本。

FlexKV 聚焦在线推理 KV 缓存核心场景，仅支持四级硬件分层存储，不提供对象存储适配能力。目前项目未并入 vLLM 主干代码，仅通过社区插件实现生态适配，稳定版本以 TCP 传输为主，RDMA 高性能能力仍在持续迭代优化。

#### 适配场景
公有云 K8s 弹性集群、批量推理业务、超长文档解析、云原生 AI 在线服务。

开源地址：https://github.com/taco-project/FlexKV

### 4. NVIDIA Dynamo｜英伟达官方推理调度与缓存编排框架
**产品定位**：NVIDIA 推出的 GPU 生态原生推理调度网关与集群编排框架，聚焦流量调度、缓存感知路由、全集群资源统一管控，于 2024 年底正式开源。

NVIDIA Dynamo 是一个开源、分布式的推理服务框架，用于在数据中心规模的多节点环境中部署模型。它支持开源推理引擎 (包括 SGLang、NVIDIA TensorRT™ LLM 和 vLLM)，通过将推理的不同阶段分离到不同的 GPU 上，把请求智能地路由到适当的 GPU 以避免冗余计算，以及通过在性价比最高的存储层级缓存数据以扩展 GPU 存储空间 (显存)，实现对分布式服务复杂运行的简化和自动化。

#### 核心技术特性
1. **PD 分离服务**：将大语言模型 (LLM) 的上下文处理阶段与生成阶段分配至不同 GPU，实现 GPU 资源的独立分配与优化，从而提升每个 GPU 服务的请求数量。
2. **LLM 感知的路由**：高效分配推理流量，通过减少因重复或重叠请求带来的高成本重计算来节省计算资源，同时确保在大型 GPU 集群中实现负载均衡。
3. **KV 缓存多级存储**：将 KV 缓存即时从有限的 GPU 内存卸载至可扩展且经济高效的存储，如 CPU 内存、本地 SSD 或网络存储。
4. **基于网络拓扑优化的 Kubernetes 服务 (Grove)**：使用统一的 Kubernetes 自定义资源，在单节点和多节点部署中实现相互依赖的 AI 推理组件的高效扩展和声明式启动顺序。
5. **GPU 规划器 (GPU Planner)**：监测分布式推理环境中的 GPU 容量，并在上下文和生成阶段动态分配 GPU 工作节点，以解决和优化性能瓶颈。
6. **低延迟通信库 (NIXL)**：加速分布式推理场景下的数据传输，同时简化不同硬件之间的传输复杂度，包括 GPU、CPU、网络和存储。
7. **AIConfigurator**：基于模型类型、GPU 预算和 SLO 定制，推荐最佳预填充 (prefill) 和解码 (decode) 的置以及模型并行策略，消除分布式部署集群参数配置的猜测和试错工作。
8. **AIPerf**：利用命令行输出和深度性能报告中的详细指标，对各类推理解决方案中的生成式 AI 模型性能进行基准测试。

其核心业务价值在于缓存感知智能路由：全局采集集群缓存分布状态，优先将请求调度至已持有对应 KV 缓存的节点，从流量调度层面大幅提升集群整体缓存命中率。

#### 适配场景
全 NVIDIA 高端 GPU 集群、企业私有化大模型部署、高并发在线对话服务。

> 名词注解：分布式推理是在多个计算设备或节点上运行 AI 模型推理的过程，通过并行计算最大限度地提高吞吐量。分布式推理允许用户根据每个工作负载的独特要求来优化延迟和吞吐量，从而提高整体性能和资源利用率。

开源地址：https://github.com/ai-dynamo/dynamo

### 5. vLLM｜业界标准推理引擎，分布式缓存依赖插件生态
vLLM 是加州大学伯克利分校开源的大模型推理加速引擎，专门解决大模型运行慢、显存不够用的问题，能让同样的显卡多处理 10 到 24 倍的请求。它通过创新的内存管理技术，让企业和个人都能用更少的硬件成本跑通大模型服务，目前已成为业界最主流的推理框架之一。

vLLM 是当前业界生态最成熟、应用最广泛的 LLM 推理引擎，其首创的 PagedAttention 分页机制已成为行业 KV Block 管理的通用标准，所有主流分布式 KVCache 系统均深度兼容该格式规范。

vLLM基于其著名的PagedAttention内存管理，已官方支持Prefill/Decode分离。在不同GPU节点间，KV Cache块可通过NCCL/RDMA直接传输。虽然不像Mooncake那样独立成池，但结合PagedAttention的块调度，vLLM本身就能搭建低延迟、高吞吐的分布式缓存流水线，适合深度定制vLLM的用户。

#### 核心突破：PagedAttention
1. **显存利用率提升至 95%+**：由于块可任意放置，vLLM 几乎能利用全部 GPU 显存。实验表明，在 A100 80GB 上运行 Llama-70B，vLLM 可支持 4x 更多并发请求。
2. **共享前缀优化**：在对话场景中，多轮对话的前缀（system prompt + 历史消息）是相同的。vLLM 允许多个请求共享同一组 KV Cache 块，避免重复计算和存储。
3. **无缝支持动态批处理（Continuous Batching）**：vLLM 引入 Continuous Batching（也称 Iteration-level Batching）：在每个解码 step，动态将新到达的请求加入当前 batch；已完成的请求立即退出，不阻塞他人；GPU 始终处于高负载状态。这使得吞吐量提升 10 倍以上，同时降低平均延迟。

缓存能力层面，vLLM 原生搭载的 APCA 自适应前缀缓存仅支持单机缓存复用，无原生分布式缓存能力。集群级跨节点 KV 共享，需通过官方 KVConnector 插件体系，对接 Mooncake、LMCache、FlexKV 等第三方分布式缓存中间件实现。

#### 适配场景
全场景通用推理服务、快速原型验证、绝大多数企业基线推理集群。

开源地址：https://github.com/vllm-project/vllm

### 6. SGLang｜Radix 前缀缓存王者，分布式架构灵活度最高
SGLang 的核心技术优势为原生 RadixAttention 前缀树机制，前缀匹配粒度更精细，天然适配分布式全局寻址场景，在多轮对话、长会话业务中缓存复用效果尤为突出。

SGLang内部采用前缀树（Radix Tree）自动管理KV Cache，天然支持相同前缀的缓存复用。其分布式运行时允许跨数据并行（DP）的多个Worker共享缓存：只要前缀相同，多个Worker可同时命中，避免重复计算。这本质上在集群内实现了一套高效的KV Cache共享机制，与SGLang框架紧密集成，开箱即用。

#### 核心突破
1. **多轮对话减少重复计算**：SGLang 用 RadixAttention 技术把对话历史的 KV 缓存像字典树一样存起来，相同前缀只算一次，多轮对话场景下缓存命中率能提升 3-5 倍，首 token 延迟降低 40% 以上。
2. **复杂生成任务编程**：提供类似写普通程序的语法，支持分支、循环、函数等，能处理多轮对话管理、带条件分支的生成、JSON Schema 约束输出、工具调用编排等复杂场景。

其内置 HiCache 仅为单机本地缓存模块，负责单实例缓存资源管理，不承担分布式全局元数据调度能力。SGLang 分布式架构无固定缓存后端绑定方案，支持自由插拔 Mooncake、LMCache 等主流分布式缓存组件，架构灵活度极高，可适配多样化业务场景。

#### 适配场景
多模态推理、Agent 智能体长会话业务、超长上下文大模型服务。

开源地址：https://github.com/sgl-project/sglang

### 学界与前沿探索方案（未大规模工业化）
1. **CacheGen**：专注KV Cache的有损/无损压缩，压缩后通过流式传输在节点间共享。既节省网络带宽，又让分离式推理的传输开销更低。已有开源原型代码。
2. **DistKV-LLM**：提出细粒度的KV块调度、放置与迁移策略，在集群中优化缓存命中率和负载均衡。部分仓库已公开原型实现，为大模型推理集群的资源调度提供了新的思考框架。

> 说明：以上方案目前适合学术研究参考，压缩传输、智能调度等思路大概率成为下一代KV Cache系统标配。

| ---- | ---- | ---- | ---- | ---- | ---- |
| Mooncake | RDMA/GDS 优先 | 全局 Radix + Block 路由 | 推荐 RDMA 组网 | 超大千卡集群 | 性能强劲、稳定性优异、架构标准化 |
| LMCache | TCP/RDMA 自适应 | 通用块寻址，前缀能力依赖上层引擎 | 无强制硬件依赖 | 中小集群 | 轻量化、无侵入、存量集群改造成本极低 |
| FlexKV | TCP 为主，RDMA 迭代中 | 分布式多级块寻址 | 通用云服务器 | K8s 弹性集群 | 云原生适配性强，闲置资源利用率高 |

### 第二类：推理/集群编排框架（自带分布式KV调度能力）
| 系统 | 传输方式 | 寻址能力 | 硬件依赖 | 集群规模 | 核心优势 |
| ---- | ---- | ---- | ---- | ---- | ---- |
| NVIDIA Dynamo | NIXL 点对点传输 | 缓存感知流量路由 | NVIDIA GPU 生态 | 中大型 GPU 集群 | 全栈硬件优化，集群调度智能高效 |

## 四、业界两种标准生产部署架构
### 方案一：极致性能架构（大厂自建算力标配）
**架构组合**：RDMA + Mooncake + vLLM/SGLang + P/D 集群分离

该方案通过物理拆分 Prefill 计算集群、Decode 推理集群与独立缓存集群，实现算力与缓存资源的隔离专属调度。Prefill 阶段生成的 KV 缓存通过 RDMA 零拷贝链路写入全局资源池，Decode 节点按需远程拉取缓存复用，彻底规避重复 Prefill 计算，有效降低首包延迟（TTFT）、提升集群整体吞吐，是超大规模、超高并发推理集群的标准落地架构。

### 方案二：极致成本架构（中小企业云原生标配）
**架构组合**：TCP + LMCache/FlexKV + 存量 vLLM 集群

该方案无需 RDMA 专用硬件，无需重构现有推理集群架构，通过插件化方式快速接入分布式缓存能力，充分盘活服务器闲置内存与磁盘资源，低成本实现全集群跨节点 KV 缓存复用。整体部署轻量化、改造成本低、性价比突出，广泛适用于中小企业云原生推理业务。

## 五、分布式 KVCache 未来演进趋势
1. **四级分层存储全面普及**：逐步形成 HBM→DRAM→远端内存→NVMe 的标准化分层存储体系，彻底打破单卡显存容量上限，实现缓存资源弹性扩容；
2. **量化缓存常态化落地**：FP8/INT8 量化 KV 缓存成为主流方案，有效降低缓存存储占用与跨节点传输带宽消耗，提升缓存部署性价比；
3. **调度与缓存深度融合**：实现流量调度、负载均衡、全局缓存寻址一体化，基于缓存分布状态智能分发请求，最大化集群全局缓存命中率；
4. **软硬件协同持续深化**：RDMA、GDS、NIXL 等硬件直通传输技术逐步替代传统 TCP 链路，进一步降低数据传输延迟，释放更多 GPU 有效算力。
5. **LLM推理基础设施标准化**：kvcache-ai生态正在将Mooncake和LMCache融合成更完整的解决方案，未来KV Cache的管理会像数据库一样，成为LLM推理基础设施的标准组件。

## 六、最终选型结论（可直接落地）
- 超大集群、极致性能、高稳定需求 → Mooncake + vLLM
- 存量集群无痛改造、低成本升级 → LMCache + vLLM
- 云原生 K8s 弹性集群、成本优先场景 → FlexKV
- 全 NVIDIA 硬件体系私有化部署 → NVIDIA Dynamo 生态
- 多模态、Agent 长会话、超长上下文业务 → SGLang + 分布式缓存后端

## 结语
分布式 KVCache 已从可选的性能优化手段，迭代为大模型云原生推理的核心基础设施。在同等算力条件下，无论是池化、分层还是解耦，这些开源系统都在让昂贵的GPU显存放出更多空间给计算，合理的分布式缓存架构，能够显著降低推理成本、压缩响应延迟、提升集群并发吞吐，是企业规模化落地大模型智能服务的核心技术壁垒与竞争力。
